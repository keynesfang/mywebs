2：Git 安装及重要概念理解

在Windows上安装Git:

首先从Git官网下载Windows版的安装程序，然后按默认选项安装即可。

Mac OS X与Linux上的Git安装方法请自行在百度与Google上搜索。

装好后，在开始菜单里找到【Git】->【Git Bash】，蹦出一个类似命令行窗口的东西，就说明Git安装成功！如下图：

在弹出的对话框中输入以下命令：

  • $ git config --global user.name "Your Name"
  • $ git config --global user.email "email@example.com"

命令行中的"Your Name"与"email@example.com"是用户自己定义的。在你的电脑中可以建立多个Git仓库， git config命令的--global参数，用了这个参数，表示你这台机器上所有的Git仓库都会使用这个配置，当然也可以对某个仓库指定不同的用户名和Email地址。到此，Git已成功部署在我们的电脑中了。

把文件夹变成Git仓库

1：在Windows操作系统上创建一个用于小说编写的专有空白目录；【为了避免遇到各种莫名其妙的问题，请确保目录名（包括父目录）不包含中文。】

2：通过git init命令把这个目录变成Git可以管理的仓库；如下图：【理论上一台电脑可以有无数个这样的独立仓库】

仓库创建好后，目录中会多出一个.git的目录，因为这个目录默认是隐藏的，如果你要看到它必须把系统中的显示隐藏文件选项打开。

【.git目录】是Git来跟踪管理版本库的，不要手动去尝试操作它，不然会把Git仓库给破坏掉的。

那么Git是怎么把我们的文件夹变成一个仓库的呢？先上一个图，再进行看图说话：

重要概念

  • 工作区：【.git目录】的父文件夹。你所有的创作工作，都是在工作区进行的。
  • 版本库：【.git目录】。它管理着你各个版本的工作成果。你可以把最新的成果提交到版本库中，也可以随时把版本库中某一版本恢复到工作区中。
  • stage（暂存区）：临时存放文件修改内容的地方。
  • master（主分支）：记录了小说相关的全部最终内容的地方。除了【主分支】外，还可以在某一时间点创建【其它分支】。比如小说完结后，很多读者对这个结局 不是很满意，这时我可以创建一个【第二结局分支】，这样大家就可以选择自己想看的结局了，皆大欢喜。【对于程序代码来说，【主分支】 可以提供一个全功能收费版的，而【其它分支】可以提供部分功能版的，以供免费用户使用。这样妈妈再也不用担心你的软件被破解了。】
  • HEAD指针：总是指向当前分支的一个标记。

重要概念的理解与消化

现在大家对Git仓库有了一个概念上的认识，但怎样才能和我们的小说创作结合起来的，请往下看：

第一天：

1. 我们在工作区新建一个文档，是一本名叫《破晓黎明》的传奇小说。

2. 开始首次的创作。完成后满意的部分存档到【stage（暂存区）】。如果存档后发现有调整的地方，还可以将暂存区中的内容还可以恢复到工作区中。

3. 觉得写到一定阶段时，可以把【stage（暂存区）】里的修改一起提交到主分支上。这时候别人就可以通过拉取你主分支上的内容进行阅读了。

第二天：

1. 小说提交到主分支后，已经有人开始在上面阅读了，为了对得起观众，从今天开始就要开始不断更新了。

2. 写了几章后觉得不错，要先提交到分支上，免得一会灵感不佳的时候乱写一通，所以想作一个版本保存。

3. 但今天的创作还没完，如果提交到主分支上，万一又有地方要修改，那就会影响到读者的心情了，所以我们要在头一天完成的主分支上创建一个【写稿】分支。

4. 从第二天开始，实际上我们都是在【写稿】分支上进行分作。等写稿分支确认无误后，再把它合并到主分支中去。所以读者看到的永远都是被精心揣摩过的内容。

第三天......第N天，都重复第二天的故事。

第N+1天：

1. 小说终于写完了，把【写稿】分支的内容最后一次合并到主分支上。

2. 可以和【写稿】分支说拜拜了，二话不说，删除【写稿】分支，这个世界清静了。

第X天：

小说完成后的某天。你突发奇想，想把很久以前某个情节的结局改动一下，来一个【第二情节版】。那你可以把版本库中找回截止到那个情节的版本， 增加一个【第二情节版】分支，然后开始再更新第二情节版的，希望告诉读者想到看【第二情节版】的，就从该分支上拉取最新的就行。世界是多么美好呀。

接下来，我们就通过第一天到第X天的情况来模拟Git的使用技巧。
